## Wat Kon Beter

### Time Management
- Documentatie nam meer tijd dan verwacht
- Acceptatietesten invullen was tijdrovend
- Laatste dag was stressvol (oplevering)

### Technical
- Achievement systeem had eerder getest kunnen worden
- Enkele edge cases pas laat ontdekt
- Theme switching had meer testing nodig

### Process
- Code review had meer aandacht kunnen krijgen
- Sommige commits waren te groot
- Documentation formatting inconsistent aanvankelijk

### Communication
- Niet altijd duidelijk wie wat deed op vrijdag
- Sommige taken overlappen

## Lessons Learned

### Wat We Geleerd Hebben
1. **Start documentatie vroeg** - Niet tot laatste moment wachten
2. **Test vaak** - Daily testing voorkomt problemen
3. **Keep it simple** - Simpele oplossingen werken het best
4. **Professionele documentatie kost tijd** - Plan hiervoor
5. **Cross-browser testing is belangrijk** - Vroeg testen

### Wat We Volgende Keer Anders Doen
1. Meer tijd reserveren voor documentatie
2. Kleinere, frequentere commits
3. Betere task tracking
4. Eerder beginnen met acceptatietesten
5. Code review als standaard proces
